Repository

Definición de la interfaz abstracta para la persistencia de datos.

Overview

Esta interfaz define el contrato que deben seguir todos los adaptadores de persistencia. Sigue el principio de inversión de dependencia para mantener el núcleo de la aplicación aislado de detalles de implementación como bases de datos o sistemas de archivos.

Methods

save(entity: Entity): void

Persiste una nueva entidad en el sistema de almacenamiento configurado.

get_by_id(id: string): Entity

Recupera una entidad específica utilizando su identificador único.

delete(id: string): boolean

Elimina una entrada del repositorio. Retorna true si la operación fue exitosa.

Data Structure

Property Type Description
id String Identificador único autogenerado (UUID).
created_at DateTime Marca de tiempo de la creación del registro.
payload Object Datos específicos de la entidad.

Usage Example

Para implementar un nuevo adaptador, hereda de esta clase en tu capa de infraestructura:

class SQLRepository(Repository): ...
