SQLiteKnowledgeRepository

Implementación de persistencia basada en SQLite para el repositorio de conocimiento.

Descripción

Esta clase extiende de KnowledgeRepository y gestiona el almacenamiento local de entradas y sus conexiones utilizando una base de datos SQLite. Soporta operaciones de Upsert, manejo de metadatos en formato JSON y relaciones con integridad referencial.

Constructor: __init__(db_path: str = "codex.db")

Inicializa la conexión y asegura la existencia de las tablas necesarias.

Esquema de Datos

La base de datos se compone de dos tablas principales:

Tabla Columnas Clave Descripción
entries id, title, content, tags, metadata Almacena el contenido principal y atributos.
relations from_id, to_id, connection_type Define grafos dirigidos entre entradas.

Métodos del Repositorio

save(entry: Entry)

Realiza un Upsert. Si el id ya existe, actualiza los campos; de lo contrario, crea un nuevo registro. Los tags y metadatos se serializan a JSON.

get_by_id(entry_id: str) -> Optional[Entry]

Busca una entrada única. Reconstruye el objeto Entry parseando los campos de fecha y los objetos JSON internos.

find_by_tag(tag: str) -> List[Entry]

Busca coincidencias parciales dentro de la columna tags utilizando patrones LIKE.

delete(entry_id: str) -> bool

Elimina la entrada y limpia automáticamente cualquier relación en la que el ID participara (evitando huérfanos).

Gestión de Conexiones

add_relation(relation: Relation)

Inserta una nueva conexión entre dos IDs. Ignora duplicados exactos (INSERT OR IGNORE).

remove_relation(relation: Relation)

Elimina el vínculo específico definido por el origen, destino y tipo de conexión.
