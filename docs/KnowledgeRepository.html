Interfaz KnowledgeRepository

Clase base abstracta (ABC) que define el contrato para la persistencia de conocimientos y sus relaciones.

Definición del Contrato

Esta interfaz asegura que cualquier adaptador de almacenamiento (SQL, NoSQL, In-Memory) proporcione los métodos necesarios para gestionar el ciclo de vida de las entidades Entry y Relation.

Métodos Requeridos

save(entry: Entry) -> None

Persiste una nueva entrada o actualiza una existente en el repositorio.

get_by_id(entry_id: str) -> Optional[Entry]

Busca y retorna una entrada específica. Devuelve None si el ID no existe.

find_by_tag(tag: str) -> List[Entry]

Recupera todas las entradas que contengan la etiqueta especificada.

delete(entry_id: str) -> bool

Elimina una entrada por su ID. Retorna True si la operación fue exitosa.

add_relation(relation: Relation) -> None

Crea un vínculo semántico entre dos entidades existentes.

remove_relation(relation: Relation) -> None

Elimina el vínculo especificado entre dos entidades.
