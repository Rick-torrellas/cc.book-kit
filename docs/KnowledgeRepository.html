KnowledgeRepository

Abstract Base Class defining the standard interface for knowledge persistence and retrieval.

Overview

The KnowledgeRepository serves as the primary abstraction layer for storing Entry objects and managing their metadata. Implementations must handle the physical storage logic (SQL, NoSQL, or Vector stores).

Methods

save(entry: Entry) -> None

Persists a new entry or updates an existing one within the repository.

get_by_id(entry_id: str) -> Optional[Entry]

Retrieves a specific entry by its unique identifier. Returns None if not found.

find_by_tag(tag: str) -> List[Entry]

Searches and returns a list of all entries associated with the specified tag.

delete(entry_id: str) -> bool

Removes an entry from the repository based on its ID. Returns True if the deletion was successful.

add_relation(relation: Relation) -> None

Establishes a link between two existing entries using a Relation object.

remove_relation(relation: Relation) -> None

Severs an existing link between entries based on the provided relation details.

Usage Example

repo = MySqlKnowledgeRepository()
entry = repo.get_by_id("12345")
if entry: print(entry.content)
